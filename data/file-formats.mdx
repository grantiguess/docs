# File Formats

Redstring uses a human-readable JSON format for its data storage. The primary file extension is `.redstring`.

## The .redstring Format

A `.redstring` file is a complete snapshot of a graph. It is designed to be **diff-friendly** for Git (pretty-printed by default).

### Structure

```json
{
  "version": "1.0.0",
  "nodes": [
    {
      "id": "node-1",
      "data": { ... },
      "position": { ... }
    }
  ],
  "edges": [
    {
      "id": "edge-1",
      "source": "node-1",
      "target": "node-2"
    }
  ],
  "transform": { "x": 0, "y": 0, "k": 1 }
}
```

## Parsing Rules

When Redstring loads a file:
1.  **Validation**: It runs a Zod schema check to ensure structural integrity.
2.  **Sanitization**: It strips unknown properties to prevent "state pollution."
3.  **Migration**: If the `version` field is older than current, it runs migration scripts (e.g., `v0.9` -> `v1.0`).

## External Assets

For nodes that reference external content (images, PDFs):
*   **Web URLs**: Stored as simple strings.
*   **Local Assets**: Stored as relative paths (e.g., `./assets/image.png`). The `UniverseBackend` resolves these to absolute paths at runtime.
