# Quick Start

Get Redstring running locally and start building your first knowledge Web in just a few minutes.

## Prerequisites

- **Node.js** 18 or higher
- **npm** or **yarn** package manager
- **Git** for version control

## Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd redstringuireact
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Start the development server**
   ```bash
   npm run dev
   ```

4. **Open your browser**
   Navigate to `http://localhost:5173` to see Redstring in action.

## Your First Knowledge Web

### Creating Things

1. **Right-click** on the canvas to open the pie menu
2. **Select "Add Thing"** to create your first concept
3. **Name your Thing** and choose a color
4. **Add a description** to provide context

### Connecting Things

1. **Click and drag** from one Thing to another to create connections
2. **Right-click connections** to add labels or modify properties
3. **Use directional arrows** to show relationships (A → B)

### Building Webs

1. **Double-click** a Thing to explore its definition Web
2. **Add related concepts** within the definition space
3. **Navigate back** using the breadcrumb navigation
4. **Create hierarchical structures** by nesting Webs within Things

## Navigation

### Canvas Controls

- **Pan**: Click and drag empty space, or use trackpad gestures
- **Zoom**: Mouse wheel, Ctrl+scroll, or pinch gestures
- **Select**: Click on Things or connections
- **Multi-select**: Ctrl+click or drag selection box

### Keyboard Shortcuts

| Action | Shortcut |
|--------|----------|
| New Thing | `N` |
| Delete | `Delete` or `Backspace` |
| Search | `Ctrl+F` |
| Save | `Ctrl+S` |
| Undo | `Ctrl+Z` |
| Redo | `Ctrl+Y` |

## Advanced Features

### Semantic Web Integration

Redstring can import knowledge from external sources:

1. **Open the Semantic Panel** (right sidebar)
2. **Search for concepts** from Wikidata, DBpedia, etc.
3. **Import knowledge clusters** to quickly build comprehensive Webs
4. **Browse relationships** and selectively add relevant connections

### File Management

- **Auto-save**: Your work is automatically saved as you go
- **Universe files**: Save complete workspaces as `.redstring` files
- **Recent files**: Quick access to recently opened projects
- **Export options**: Share your knowledge Webs in various formats

### Collaboration

- **GitHub Integration**: Sync your knowledge Webs with GitHub repositories
- **Version control**: Track changes and collaborate with others
- **OAuth authentication**: Secure access to external services

## Development Commands

If you're working on Redstring itself:

```bash
# Run development server
npm run dev

# Run tests
npm test

# Build for production
npm run build

# Generate documentation
npm run docs:generate

# Serve documentation locally
npm run docs:dev

# Run with all services
npm run dev:full
```

## Configuration

### Environment Variables

Create a `.env` file in the project root:

```env
# GitHub App credentials (optional)
GITHUB_APP_ID=your_app_id
GITHUB_PRIVATE_KEY=your_private_key

# OAuth configuration (optional)
OAUTH_CLIENT_ID=your_oauth_client_id
OAUTH_CLIENT_SECRET=your_oauth_client_secret

# API endpoints (optional)
SPARQL_ENDPOINT=https://query.wikidata.org/sparql
```

### Customization

Redstring is highly customizable:

- **Colors**: Modify the theme in `src/styles/`
- **Keybindings**: Edit keyboard shortcuts in `src/hooks/useKeyboardShortcuts.js`
- **Canvas behavior**: Adjust pan/zoom sensitivity in `src/NodeCanvas.jsx`
- **Data sources**: Add new semantic web endpoints in `src/services/`

## Troubleshooting

### Common Issues

**Canvas not responding to input**
- Check that JavaScript is enabled
- Try refreshing the page
- Ensure you're not in a nested iframe

**Slow performance with large Webs**
- Use the abstraction controls to hide complexity
- Break large Webs into smaller, focused subgraphs
- Clear browser cache if issues persist

**Semantic web queries failing**
- Check your internet connection
- Verify SPARQL endpoints are accessible
- Some endpoints have rate limiting
