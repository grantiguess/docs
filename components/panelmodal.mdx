# Unified Panel System

Redstring avoids cluttering the canvas with floating toolbars. Instead, all heavy interactions happen in the **Unified Panel**.

## Architecture

The Panel is a modal overlay that can exist in three states:
1.  **Hidden**: Completely removed from DOM.
2.  **Docked**: Pinned to the side of the screen.
3.  **Floating**: Draggable window over the canvas.

### Context Awareness

The Panel content changes dynamically based on **Selection Context**:

*   **No Selection**: Shows Global Search & Settings.
*   **Single Node**: Shows Node Inspector (Properties, Text Editor).
*   **Multiple Nodes**: Shows Bulk Actions (Align, Color, Group).

## Key Components

### `PanelModal`
The container wrapper that handles the window chrome (header, resize handles, drag logic).

### `InternalPanelRouter`
A switch-case component that renders the correct "View" based on the store state.

*   `LeftHistoryView`: Undo/Redo stack visualization.
*   `LeftLibraryView`: Asset browser.
*   `NodeProperties`: The semantic editor for a node.

## Design Philosophy

We use a **"Command Center"** philosophy. The user works freely on the canvas, then "retreats" to the Panel for deep work (writing long notes, tweaking metadata), then returns to the canvas. This separates "Thinking" (Canvas) from "Defining" (Panel).
