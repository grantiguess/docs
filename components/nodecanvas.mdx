# NodeCanvas

The `NodeCanvas` is the heart of Redstring. It is more than just a `div`; it is a complex interaction engine handling gestures, virtualization, and coordinate systems.

## Key Features

1.  **Infinite Pan/Zoom**: Powered by D3-zoom behavior with smooth inertial damping.
2.  **Virtualization**: Uses `react-window` concepts (custom implementation) to cull nodes outside the viewport, maintaining 60fps even with 10k+ items.
3.  **Coordinate Systems**: Manages the translation between "Screen Space" (pixels on your monitor) and "Graph Space" (logical coordinates).

## Interaction Model

### Navigation
*   **Pan**: Middle-click drag OR Space + Left-click drag.
*   **Zoom**: Mouse wheel.
*   **Fit View**: Double-tap spacebar centered the view on content.

### Manipulation
*   **Selection**: Click to select. Shift+Click to add to selection.
*   **Marquee**: Click-drag on empty space creates a selection box.
*   **Transform**: Drag interactions are passed to the `GraphStore` which updates coordinates via Immer for performance.

## The Rendering Loop

To maintain performance, the Canvas separates **Layout** from **Rendering**.

1.  **Layout (Store)**: React state holds the logic.
2.  **Rendering (CSS/SVG)**: We use CSS Transforms for node positioning (GPU accelerated) and SVG for edges (vector precision).

> [!NOTE]
> **Optimization**: We intentionally avoid React Re-renders for drag operations. Instead, we use refs and direct DOM manipulation for the "dragging" frame, only syncing to the store on `dragEnd`.
